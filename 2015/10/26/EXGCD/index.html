辗转相除法

千年之前，业界毒瘤 Ευκλειδης（欧几里得），发明了欧几里得算法（辗转相除法），让我们先为这位先辈默哀3分钟。。。。。。

设c=gcd(m,n), d=gcd(n,b)

　　1.　∵c为m和n的公约数

　　　　∴m能被c整除，n也能被c整除

　　　　∴na也能被c整除 参照推论一

　　　　∴m-na也能被c整除(即b能c整除) 参照推论二

　　　　∴c为n和b的公约数

　　　　∵d为n和b的最大公约数

　　　　∴c≤d

　　2.　同理可证 d≤c

　　　　∵d为n和b的公约数

　　　　∴n能被d整除，b也能被d整除

　　　　∴na也能被d整除 参照推论一

　　　　∴na+b也能被d整除(即m能d整除) 参照推论二

　　　　∴d为m和n的公约数

　　　　∵c为m和n的最大公约数

　　　　∴d≤c

综上所述：c=d，即gcd(m,n)=gcd(n,r) .□

当然中国的九章算术也提出了更相减损术，在这里就不再介绍了。

扩展欧几里得算法

扩展欧几里得算法是算法竞赛，也是noip的一项重点内容。
对于任意正整数 a , b;
总有
a x1 + b y1 = gcd(a, b);
b x2 + a % b y2 = gcd(b, a % b);
∵ gcd(a + b) = gcd(b + a % b);
∴ a x1 + b y1 = b x2 + (a - (a / b) b) y2;
注：(c++ 中 a % b = a - (a / b) - b)
整理得 x1 = y2 , y1 = x2 - (a / b) y2;

Problem

1

总时间限制: 1000ms 内存限制: 65536kB
描述:
给定正整数a，b，c。求不定方程 ax+by=c 关于未知数x和y的所有非负整数解组数。

输入
一行，包含三个正整数a，b，c，两个整数之间用单个空格隔开。每个数均不大于1000。
输出
一个整数，即不定方程的非负整数解组数。
样例输入:
2 3 18
样例输出:
4

首先这道题数据范围暴力是完全可以跑出来的，但如果遇到变态的出题人就会毫不犹豫的卡掉你，虽然不知道这是不是最优算法，但下面推导的要优化好多。

  1. 当 c ≡ k （mod gcd（a, b）)， k ≠ 0时;
    显然解的组数为零；
  2. 当 gcd(a, b) | c 时；
    原方程的一组整数解为
    x1 = x (c / d), y1 = x (c / d);
    则所有的整数解可以表示为
    Xn = x1 + (b / d) t, Yn = y1 + (a / d) t, t ∈ Z；
    ∵ Xn >= 0, Yn >= 0
    ∴ t ∈（-x d / b, d y / a）;

2

描述
求关于x的同余方程ax ≡ 1 (mod b)的最小正整数解。
格式
输入格式

输入只有一行，包含两个正整数a, b，用一个空格隔开。
输出格式

输出只有一行，包含一个正整数x0，即最小正整数解。输入数据保证一定有解。
样例1
样例输入:

3 10
样例输出:

7
限制
每个测试点1s
提示
对于40%的数据，2 ≤b≤ 1,000；
对于60%的数据，2 ≤b≤ 50,000,000；
对于100%的数据，2 ≤a, b≤ 2,000,000,000

对于这题简单的EXGCD， 原题可转化为 ax - by = 1, 因为减号不影响，改为加号就好了， 最后求出一组解，x = (x + b) % b, 得出最小解。
